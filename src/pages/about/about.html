Documentation

This app basically helps faculties to manage their attendence list in an efficient way.
  • The app first routes the user to a login page, where the user needs to input their email-id and password. (for testing use password as login123).
  • After successful login the app routes to tabs page, which contains 4 tab sections:
    1. List tab
    2. Add tab
    3. Settings tab
    4. About tab
  • By default, the app routes the user to list(attendence-list) page, where the user can take attendence and/or scroll down to view attendence details.
  • When the user wishes to take attendence, user must choose the date for which the attendence is being taken, by default the date is set to current date. On clicking TAKE ATTENDENCE button, user is routed to Mark attendence page.
  • Mark Attendence page contains the list of students with respective toggle buttons. If the toggle is switched to green then, the student is marked present, otherwise absent. On clicking submit, the user is redirected back to attendence-list page.
    Note: The records remain uneffected till the submit button is tapped.
  • To add new student into the data base switch to AddAdd-Student tab, fill the form appropriately, and tap Add to records.
  • The settings tab helps the user to manage the database easily by having two user friendly buttons:
    1. Clear records
    2. Populate records
  • Clear records button clears all the records stored in database.
    Note: this action can not be reversed, procced with caution.
  • Populate records button generates 50+ dummy records in database.
    Used for testing purposes only!
  • About tab contains all the app necessary details. If you need more documentation send a request to ksaikiranr@gmail.com
